

Excellent question 👏

In **Next.js (App Router)** — handling a **global 404 page** is super easy and elegant.
Here’s the **right and modern** way to do it 👇

---

## ✅ 1️⃣ Create a `not-found.tsx` file at the root of your `app/` directory

That’s all you need.
Next.js automatically uses that page for **all unmatched routes**.

**📁 Folder structure:**

```
app/
 ├─ (public)/
 ├─ (admin)/
 ├─ layout.tsx
 ├─ globals.css
 └─ not-found.tsx   ← this handles 404 globally
```

---

## ✅ 2️⃣ Example `not-found.tsx` code

```tsx
import Link from "next/link";

export default function NotFound() {
  return (
    <div className="min-h-screen flex flex-col items-center justify-center bg-gray-100 text-center px-6">
      <h1 className="text-6xl font-bold text-purple-700 mb-4">404</h1>
      <h2 className="text-2xl font-semibold text-gray-800 mb-2">
        Page Not Found
      </h2>
      <p className="text-gray-600 mb-6">
        Oops! The page you’re looking for doesn’t exist or has been moved.
      </p>
      <Link
        href="/"
        className="bg-purple-700 text-white px-6 py-3 rounded-md hover:bg-purple-800 transition"
      >
        Go Back Home
      </Link>
    </div>
  );
}
```

---

## ✅ 3️⃣ Optional — Custom 404 for specific groups

If you’re using route groups like:

```
app/
 ├─ (public)/
 │   └─ not-found.tsx
 ├─ (admin)/
 │   └─ not-found.tsx
```

Each group can have its **own** `not-found.tsx` (for example, admin could have a different message).

---

## ✅ 4️⃣ Trigger it manually (optional)

If you want to **programmatically show** a 404 page from a server component or route handler:

```tsx
import { notFound } from "next/navigation";

export default function Page({ params }) {
  const data = getData(params.id);

  if (!data) {
    notFound(); // redirects to app/not-found.tsx
  }

  return <div>{data.name}</div>;
}
```

---

### 🔥 Summary

| Location                    | File   | Purpose                             |
| --------------------------- | ------ | ----------------------------------- |
| `app/not-found.tsx`         | Global | Handles all 404 routes              |
| `app/(group)/not-found.tsx` | Scoped | Handles 404 within that route group |
| `notFound()` function       | Code   | Programmatically trigger a 404      |
